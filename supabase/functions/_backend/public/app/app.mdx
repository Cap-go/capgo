Apps are the foundational entities in Capgo. Each app represents a unique Capacitor application that you can manage and update through the platform. The Apps API allows you to create, retrieve, update, and delete app configurations.

## Understanding Apps

An app in Capgo represents your Capacitor application and includes:
- **App ID**: Unique identifier for your application
- **Name**: Human-readable name of your application
- **Icons**: Visual identifiers for your app in the dashboard
- **Configuration**: Settings that control how updates are delivered
- **Ownership**: Organization and user access information
- **Usage Statistics**: Metrics about installs and updates

## Best Practices

1. **Naming Convention**: Use clear, identifiable names for your apps
2. **Security**: Protect your API keys and access credentials
3. **Organization**: Group related apps under the same organization
4. **Monitoring**: Regularly check app statistics and performance
5. **Backup**: Maintain configuration backups for critical apps

## Endpoints

### GET

`https://api.capgo.app/app/`

Retrieve information about your apps.

#### Query Parameters
- `id`: Optional. Specific app ID to retrieve
- `page`: Optional. Page number for pagination
- `limit`: Optional. Number of results per page (default: 50)

#### Response Type

```typescript
interface App {
  app_id: string
  created_at: string | null
  default_upload_channel: string
  icon_url: string
  id: string | null
  last_version: string | null
  name: string | null
  owner_org: string
  retention: number
  transfer_history: Json[] | null
  updated_at: string | null
  user_id: string | null
}
```

#### Example Request

```bash
# Get all apps
curl -H "authorization: your-api-key" \
  "https://api.capgo.app/app/"

# Get specific app
curl -H "authorization: your-api-key" \
  "https://api.capgo.app/app/?id=app_123"
```

#### Example Response

```json
{
  "data": [
    {
      "id": "app_123",
      "name": "My Awesome App",
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z",
      "icon_url": "https://example.com/icon.png",
      "owner_org": "org_123",
      "external_id": null,
      "dev_count": 5,
      "prod_count": 1000,
      "last_activity": "2024-01-10T00:00:00Z",
      "storage_used": 25000000,
      "retention_policy": 30,
      "custom_id": "my-awesome-app",
      "user_id": "user_123"
    }
  ]
}
```

### POST

`https://api.capgo.app/app/`

Create a new app.

#### Request Body

```typescript
interface AppCreate {
  name: string
  icon_url?: string
  owner_org: string
}
```

#### Example Request

```bash
# Create new app
curl -X POST \
  -H "authorization: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My New App",
    "icon_url": "https://example.com/icon.png",
    "owner_org": "org_123"
  }' \
  https://api.capgo.app/app/
```

#### Success Response

```json
{
  "id": "app_123",
  "name": "My New App",
  "status": "ok"
}
```

### PUT

`https://api.capgo.app/app/`

Update an existing app.

#### Request Body

```typescript
interface AppPut {
  id: string
  name?: string
  icon?: string
  retention?: number
}
```

#### Example Request

```bash
curl -X PUT \
  -H "authorization: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "app_123",
    "name": "Updated App Name",
    "icon": "https://example.com/updated-icon.png",
    "retention": 45
  }' \
  https://api.capgo.app/app/
```

#### Success Response

```json
{
  "id": "app_123",
  "name": "Updated App Name",
  "status": "ok"
}
```

### DELETE

`https://api.capgo.app/app/`

Delete an app and all associated resources. Use with extreme caution as this action cannot be undone.

#### Request Body

```typescript
interface AppDelete {
  id: string
}
```

#### Example Request

```bash
curl -X DELETE \
  -H "authorization: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "app_123"
  }' \
  https://api.capgo.app/app/
```

#### Success Response

```json
{
  "status": "ok"
}
```
